二、整体结构设计

1、整体结构

整个项目的核心为工作线程，对应于src/net/src/exmsg.c模块。这个模块是TCP/IP核心线程通信模块，运行了一个核心线程，所有 TCP/IP 中相关的事件都交由该线程处理。

为什么在协议栈初始化时需要先进行硬件资源初始化？是为了适应不同平台吗？

在编写 TCP/IP 协议栈的过程中，在协议栈初始化时首先进行硬件资源初始化是非常重要的一步，原因主要有以下几个方面：

  1. 适应不同平台：不同的平台可能有不同的硬件资源和配置要求。进行硬件资源初始化可以确保协议栈能够适应不同的平台，并能够正确地与底层硬件进行交互。
  2. 确保硬件资源可用：协议栈的运行依赖于底层的网络硬件资源（如网卡、网络接口等）。在协议栈初始化时，首先初始化硬件资源可以确保这些资源是可用的，并且已经准备好进行数据传输。
  3. 设置硬件参数：硬件资源初始化过程通常包括设置硬件参数（如网卡的 MAC 地址、IP 地址、子网掩码等）。这些参数对于协议栈的正确运行至关重要，因为它们决定了如何在网络中进行数据传输。
  4. 注册硬件中断和回调函数：硬件资源初始化过程中通常需要注册硬件中断和回调函数，以便在数据包到达时能够及时通知协议栈进行处理。如果没有进行硬件资源初始化，协议栈将无法及时响应网络事件。
  5. 分配和管理资源：硬件资源初始化过程可能涉及到分配和管理一些必要的资源，如内存缓冲区、DMA 通道等。这些资源对于协议栈的高效运行也是至关重要的。
  6. 错误处理：在硬件资源初始化阶段，可以检测和处理可能的硬件错误。如果硬件资源不可用或者初始化失败，可以在这个阶段进行相应的错误处理，避免后续协议栈运行时出现难以调试的问题。

当应用程序需要请求工作线程去做某事时，需要先申请一个消息结构，即一个内存块，并将这个内存块的指针插入消息队列中。工作线程在接收到这个指针后执行相应的操作，完成后释放该内存块。

本协议栈在设计时采用固定长度的存储分配，考虑以下两点：

1、在某些系统上，内核并没有提供动态内存分配函数；– 考虑可移植性。

2、减少动态分配造成的内存碎片问题。

在初始化时，将数组中各个数组元素构建成一个空闲链表，当需要内存时，从中取出一个内存块用于分配，并且进行加锁处理，以适用于多线程场景。

2、mblock.c

定长内存块管理器（Fixed-size Memory Block Manager）通过将一个连续的内存区域分割成固定大小的块，然后使用链表管理这些块，实现了高效的内存分配和释放。以下是该模块的详细分析：

结构体定义

  • free_list: 空闲块链表，用于存储空闲内存块。
  • start: 内存块起始地址，目前未使用。
  • locker: 互斥锁 nlocker_t，用于保护对 free_list 的访问，确保线程安全性。
  • alloc_sem: 分配信号量 sys_sem_t，用于控制在没有空闲块时的等待。

初始化函数 mblock_init

该函数用于初始化定长内存块管理器：

  1. 将传入的内存区域 mem 按照 blk_size 划分为 cnt 个固定大小的内存块，并将这些块使用链表 free_list 连接起来。
  2. 初始化互斥锁 locker，用于保护对 free_list 的并发访问。
  3. 如果 locker_type 表示需要多线程支持，就创建分配信号量 alloc_sem，用于在没有空闲块时的等待。

分配函数 mblock_alloc

该函数用于从管理的内存块中分配一个空闲块：

  1. 如果 ms < 0 或者管理器未使用锁（即 locker.type == NLOCKER_NONE），则直接查询空闲块数量。
  2. 如果有等待信号量，使用 sys_sem_wait 等待空闲块的释放。
  3. 使用互斥锁保护对 free_list 的访问，从链表头部取出一个空闲块。
  4. 返回分配的空闲块地址。

空闲块数量查询函数 mblock_free_cnt

该函数用于获取当前空闲块的数量：

  1. 使用互斥锁保护对 free_list 的访问，获取链表中当前空闲块的数量。
  2. 返回空闲块数量。

释放函数 mblock_free

该函数用于释放先前分配的空闲块：

  1. 使用互斥锁保护对 free_list 的访问，将要释放的空闲块插入到链表的尾部。
  2. 如果存在等待信号量，使用 sys_sem_notify 通知等待的线程有新的空闲块可用。

销毁函数 mblock_destroy

该函数用于销毁内存块管理器：

  1. 如果存在分配信号量 alloc_sem，释放该信号量。
  2. 销毁互斥锁 locker。
  3. 注意：该函数不会释放内存块本身，需要调用者负责管理 mem 的释放。

总结

定长内存块管理器通过互斥锁和信号量保证了在多线程环境下的安全操作，有效地管理和利用固定大小的内存块，避免了内存碎片和资源竞争问题。每个操作都经过严格的同步控制，确保了对内存块的分配和释放的正确性和高效性。

五、定长消息队列的实现

1、协议栈中涉及定长消息队列的部分

消息队列主要用在两个地方：

  1. 工作线程，接收来自应用程序或网卡的消息；
  2. 网卡部分，涉及输入队列和输出队列，放置需要发送的数据包。

2、fixq.c

定长消息队列的实现在多线程环境中非常关键，它能有效地管理有限的资源，避免资源耗尽和竞争条件。让我们来分析一下定长消息队列的具体实现：

结构体定义

  • size: 消息队列的大小，即可以容纳的消息数量。
  • in, out: 写入和读取消息的索引位置，循环使用以节约空间。
  • cnt: 当前消息队列中的消息个数。
  • buf: 指向存储消息的数组。
  • locker: 用于保护消息队列的互斥锁。
  • recv_sem, send_sem: 分别用于控制读和写的信号量。

初始化函数 fixq_init

该函数用于初始化定长消息队列：

  1. 初始化互斥锁 locker，确保在多线程环境下对队列操作的原子性和互斥性。
  2. 创建两个信号量 recv_sem 和 send_sem，用于控制读和写操作的同步，确保队列空间和消息的正确使用。
  3. 将外部传入的消息缓冲区 buf 和队列大小 size 赋值给消息队列结构体。

发送函数 fixq_send

该函数用于向消息队列中发送消息 msg：

  1. 首先通过互斥锁 locker 确保对队列操作的互斥性。
  2. 判断消息队列是否已满，如果满且超时参数 tmo 小于 0，则返回 NET_ERR_FULL 表示队列已满且不等待。
  3. 使用写信号量 send_sem 等待有空闲单元可以写入。
  4. 通过互斥锁再次确认队列未满，将消息写入队列，并更新索引和计数器。
  5. 发送完毕后，通过读信号量 recv_sem 通知其他线程有新消息可读取。

接收函数 fixq_recv

该函数用于从消息队列中接收消息：

  1. 首先通过互斥锁 locker 确保对队列操作的互斥性。
  2. 如果队列为空且超时参数 tmo 小于 0，则立即返回 NULL 表示超时或队列为空。
  3. 使用读信号量 recv_sem 等待有消息可读。
  4. 通过互斥锁再次确认队列非空，从队列中读取消息，并更新索引和计数器。
  5. 接收完毕后，通过写信号量 send_sem 通知其他线程有空闲空间可以写入新消息。

销毁函数 fixq_destroy

该函数用于销毁消息队列：

  1. 销毁互斥锁 locker。
  2. 释放读信号量 recv_sem 和写信号量 send_sem。
  3. 不会释放消息缓冲区 buf，因为它是外部传入的，由外部管理。

计数函数 fixq_count

该函数用于获取当前消息队列中的消息个数：

  1. 使用互斥锁 locker 锁定，获取 cnt 计数器的值。
  2. 返回当前消息个数。

总结

定长消息队列通过互斥锁和信号量确保了多线程环境下对队列操作的安全性和同步性，避免了竞争条件和资源冲突。每个操作函数都在操作前后使用互斥锁进行保护，并利用信号量进行同步控制，保证了队列的正确操作顺序和状态。

定长消息队列依赖定长内存块来管理消息的存储。每当需要发送一个消息时，消息队列会向内存块管理器申请一个空闲的内存块来存储该消息。当消息处理完毕后，消息队列会将该内存块释放回内存块管理器，以便再次利用。

初始化

初始化函数 pktbuf_init() 主要用于初始化内存块和数据包缓冲区的空闲列表，并初始化锁。它调用 mblock_init 函数对空闲块列表和空闲包列表进行初始化。

数据块管理

分配数据块

分配数据块的函数 pktblk_alloc()，从空闲块列表中获取一个数据块，并初始化其数据指针和大小。

释放数据块

释放数据块的函数 pktblk_free()，将数据块放回空闲块列表。

数据包管理

分配数据包

分配数据块的函数 pktblk_alloc()，从空闲块列表中获取一个数据块，并初始化其数据指针和大小。

释放数据包

释放数据块的函数 pktbuf_free()，将数据块放回空闲块列表。

函数 pktbuf_resize(pktbuf_t *buf, int to_size) 用于调整数据包的大小。

数据包合并

函数 pktbuf_join(pktbuf_t *dst, pktbuf_t *src) 用于将源数据包 src 合并到目标数据包 dst 中。

八、以太网协议

在协议栈中增加对以太网协议的支持，通过网卡收发数据包。

1、基础

以太网协议的主要功能：

  1. 发送上层传来的 IP 数据包到相邻计算机；
  2. 网卡接收数据包，提取出的 IP 数据包上传到上层。

以太帧有很多类型，不同类型有不同的格式和 MTU 值，最常见的是 Ethernet ||，这也是本项目采用的帧类型。

2、设计步骤

  1. 在已经添加环回接口的基础上，使用 pcap 接口添加以太网接口到协议栈中。
  2. 在以太网接收线程中接收数据包并写入 pktbuf 中，然后发消息给工作线程。
  3. 将从以太网接收到的数据包做简单处理后，通过网络接口发送出去，以测试基本的数据收发流程。
  4. 工作线程在收到链接层的数据包之后，需要决定如何处理。不同的链路层有不同的处理方式吗，工作线程需要根据不同协议灵活处理。
  5. 在收到链路上的以太网包后，协议栈只需要对包进行简单的检查，移动以太网包头，然后提交给 IP/ARP 协议进行处理。为减少工作线程接收到的消息数据，只有在消息队列为空时收到数据后，才给工作线程发消息。
